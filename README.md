# Biotech-Analyzer v3.0

Biotech-Analyzer v3.0 is a streamlined, AI-driven investment intelligence platform purpose-built for the biotechnology sector. It replaces a sprawling legacy system with a lean, containerized stack designed for maximum leverage and minimal complexity. The platform empowers retail investors with institutional-quality analysis by automating the discovery and monitoring of biotech companies, scrutinizing clinical and financial data, and surfacing strategy-specific recommendations.

## 🚀 Key Features

- **The Agent Council (CrewAI):** 10 specialized AI agents working across 3 hierarchical crews (Data Collection, Analysis, and Strategy) to execute complete investment analyses.
- **100% Local GPU Inference:** Powered by Ollama on dual Tesla P4 GPUs. No cloud models—ensuring zero inference cost and complete data privacy.
- **Biotech-Specific Intelligence:** Dedicated agents handle clinical trial statistical validation (Peer Reviewer), SEC EDGAR insider tracking (Insider), FDA PDUFA calendar scanning (Oracle), and Cash-Secured Put strike selection (Volatility).
- **Local Multi-Notebook RAG:** A self-hosted document retrieval system that partitions documents (like 10-Ks) to bypass arbitrary context limits.
- **Ticker-First Onboarding Pipeline:** A robust 7-step onboarding process mapping companies via 10-K extractions rather than fragile entity name matching.
- **Web Dashboard:** A Python-native Streamlit interface for visualizing portfolio performance, agent memos, and active signals.

## 🏗️ System Architecture

- **Agent Orchestration:** [CrewAI](https://crewai.com/)
- **Local LLMs:** Ollama (`llama3.1:8b` for parsing, `llama3.2:3b` for low-latency queries, `deepseek-r1:7b` for reasoning)
- **Knowledge Base:** Local Multi-Notebook RAG (ChromaDB)
- **Transactional DB:** SQLite (WAL mode) & SQLModel ORM
- **UI & Deployment:** Streamlit & Docker Compose
- **Data Sources:** E*TRADE API, Yahoo Finance, ClinicalTrials.gov API v2, AACT DB, SEC EDGAR, FDA Orphan Drug DB, FDA Orange/Purple Books.

For a detailed breakdown of the system components, data sources, and dataflows, please review the documentation in the `/docs` directory.

## 🛠️ Deployment

The entire software stack is orchestrated via a single `docker-compose.yml` file, maintaining environment parity across development and production servers.

```bash
docker-compose up -d
```

This starts the Streamlit app, Ollama instances, ChromaDB, and the APScheduler. API keys for financial data providers run through isolated environment variables via the `.env` file.
